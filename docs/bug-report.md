# Bug 单

## BUG-001 注册接口未校验用户名长度，短用户名可注册成功

| 字段 | 内容 |
| --- | --- |
| Bug 编号 | BUG-001 |
| 标题 | 注册接口未校验用户名长度，短用户名可注册成功 |
| 严重程度 | Major |
| 优先级 | P1 |
| 状态 | 新建 |
| 发现日期 | 2026-07-07 |
| 发现阶段 | 接口自动化测试、页面手工测试 |
| 关联需求 | PRD-FR-003 |
| 关联规则 | REG-002 |
| 关联验收标准 | AC-003 |
| 关联测试用例 | TC-REG-003 |
| 影响接口 | `POST /api/register` |
| 影响页面 | `/register` |

## 1. 前置条件

1. 后端服务已启动。
2. 前端服务已启动。
3. 用户名 `abc` 未被注册。

## 2. 复现步骤

### 2.1 页面复现

1. 打开 `http://127.0.0.1:5173/register`。
2. 在用户名输入框输入 `abc`。
3. 在密码输入框输入 `Password123`。
4. 在确认密码输入框输入 `Password123`。
5. 点击注册按钮。

### 2.2 接口复现

请求接口：

```text
POST /api/register
```

请求体：

```json
{
  "username": "abc",
  "password": "Password123",
  "confirm_password": "Password123"
}
```

## 3. 期望结果

注册失败，返回错误提示：

```json
{
  "success": false,
  "message": "用户名长度不能少于6位",
  "data": null
}
```

## 4. 实际结果

注册成功，返回：

```json
{
  "success": true,
  "message": "注册成功",
  "data": {
    "username": "abc"
  }
}
```

## 5. 影响范围

1. 注册接口未满足 PRD-FR-003。
2. 注册页面允许不符合规则的用户名进入系统。
3. SQLite `users` 表中会保存不符合业务规则的数据。

## 6. 修复建议

在后端 `POST /api/register` 中增加 `username` 长度校验：

1. 当 `len(username) < 6` 时返回 400。
2. 响应信息为 `用户名长度不能少于6位`。
3. 保持前端也可以增加同样校验，但以后端校验作为最终规则。
