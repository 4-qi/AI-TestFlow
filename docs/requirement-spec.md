## 2. 系统边界

本系统只实现登录注册 Demo 所需的最小功能，不包含邮箱验证、短信验证、找回密码、权限管理、多角色管理、后台管理等扩展能力。

系统由前端、后端和 SQLite 数据库组成：

1. 前端负责页面展示、表单输入、接口调用和提示反馈。
2. 后端负责参数校验、用户数据读写、密码校验和接口响应。
3. SQLite 负责保存用户账号数据。

## 3. 功能模块拆解

## 6. 接口规格

### 6.1 注册接口

接口路径：`/api/register`

请求方法：`POST`

请求体：

```json
{
  "username": "testuser",
  "password": "Password123",
  "confirm_password": "Password123"
}
```

成功响应：

```json
{
  "success": true,
  "message": "注册成功",
  "data": {
    "username": "testuser"
  }
}
```

失败响应：

```json
{
  "success": false,
  "message": "用户名不能为空",
  "data": null
}
```

校验规则：

| 规则编号 | 规则描述 | 失败提示 |
| --- | --- | --- |
| REG-001 | `username` 不能为空 | 用户名不能为空 |
| REG-002 | `username` 长度必须大于等于 6 位 | 用户名长度不能少于6位 |
| REG-003 | `password` 不能为空 | 密码不能为空 |
| REG-004 | `confirm_password` 必须与 `password` 一致 | 两次输入的密码不一致 |
| REG-005 | `username` 不能重复 | 用户名已存在 |

预埋缺陷要求：

后端注册接口保留 REG-002 校验缺失问题，用于测试阶段发现 PRD-FR-003 未被实现。

### 6.2 登录接口

接口路径：`/api/login`

请求方法：`POST`

请求体：

```json
{
  "username": "testuser",
  "password": "Password123"
}
```

成功响应：

```json
{
  "success": true,
  "message": "登录成功",
  "data": {
    "username": "testuser"
  }
}
```

失败响应：

```json
{
  "success": false,
  "message": "用户名或密码错误",
  "data": null
}
```

校验规则：

| 规则编号 | 规则描述 | 失败提示 |
| --- | --- | --- |
| LOGIN-001 | `username` 不能为空 | 用户名不能为空 |
| LOGIN-002 | `password` 不能为空 | 密码不能为空 |
| LOGIN-003 | 用户名必须存在 | 用户名或密码错误 |
| LOGIN-004 | 密码必须正确 | 用户名或密码错误 |

### 6.3 当前用户接口

接口路径：`/api/me`

请求方法：`GET`

成功响应：

```json
{
  "success": true,
  "message": "获取当前用户成功",
  "data": {
    "username": "testuser"
  }
}
```

未登录响应：

```json

  "success": false,
  "message": "用户未登录",
  "data": null
}
```

### 6.4 退出登录接口

接口路径：`/api/logout`

请求方法：`POST`

成功响应：

```json
{
  "success": true,
  "message": "退出登录成功",
  "data": null
}
```

## 7. 页面规格

### 7.1 注册页面

页面路径：`/register`

页面元素：

1. 用户名输入框。
2. 密码输入框。
3. 确认密码输入框。
4. 注册按钮。
5. 跳转登录入口。

提交行为：

1. 点击注册按钮后调用 `/api/register`。
2. 注册成功后提示注册成功。
3. 注册成功后跳转 `/login`。
4. 注册失败后展示后端返回的错误提示。

### 7.2 登录页面

页面路径：`/login`

页面元素：

1. 用户名输入框。
2. 密码输入框。
3. 登录按钮。
4. 跳转注册入口。

提交行为：

1. 点击登录按钮后调用 `/api/login`。
2. 登录成功后保存登录状态。
3. 登录成功后跳转 `/home`。
4. 登录失败后展示后端返回的错误提示。

### 7.3 首页

页面路径：`/home`

页面元素：

1. 当前登录用户名。
2. 退出登录按钮。

页面行为：

1. 未登录访问时跳转 `/login`。
2. 点击退出登录后清除登录状态并跳转 `/login`。
